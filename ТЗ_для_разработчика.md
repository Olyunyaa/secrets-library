# Интеграция онбординга и библиотеки в Sailbot

## Как это должно работать

```
Участник оплачивает клуб / вступает с программы Тани
        ↓
Sailbot открывает доступ (участник = активный)
        ↓
В меню бота появляются две кнопки:
   [Онбординг и Библиотека]  ← запускает сценарий онбординга
   [Библиотека]               ← открывает Mini App напрямую
        ↓
Нажатие "Онбординг и Библиотека" → запускается наш сценарий (4 вопроса → дорожная карта → drip)
```

Неактивные участники этих кнопок не видят.

---

## Что делает Вова

### 1. Добавить кнопку «Библиотека» для всех активных участников
Открывает Mini App: `https://olyunyaa.github.io/secrets-library/app.html`
(Telegram WebApp button, без параметров — показывает всю библиотеку)

### 2. Добавить кнопку «Онбординг и Библиотека» для активных участников
Нажатие запускает сценарий онбординга (то, что в нашем боте делает `/start`).

### 3. Реализовать сценарий онбординга
Рабочий пример: [@SecretsLibraryBot](https://t.me/SecretsLibraryBot) — можно потыкать.
Исходный код: `bot.py` в репо `github.com:Olyunyaa/secrets-library.git`

**Шаг 1 — Q1:** «Как бы вы описали себя сегодня?» (мульти-выбор, кнопка Готово)
- В найме и хочу начать своё
- В найме + свои консультации/проект
- Фрилансер или предприниматель
- Хочу личный бренд и расти в соцсетях
- Пока изучаю и присматриваюсь

**Шаг 2 — Q2:** «На что хотите взять фокус в клубе?» (мульти-выбор, Готово)
- Как строить портфель без прикрас → `pain_1_portfolio_raw`
- Начать портфельную карьеру → `pain_2_portfolio_start`
- Создать проект для портфеля → `pain_3_project_start`
- Построить личный бренд → `pain_4_personal_brand`
- Соцсети и блог для портфеля → `pain_5_social_blog`
- Не умею / боюсь продавать → `pain_6_selling_fear`
- Не знаю, кому продавать → `pain_7_audience`
- Свой большой проект → `pain_8_big_project`
- Страхи, прокрастинация, саботаж → `pain_9_blockers`
- Примеры других и окружение → `pain_10_community`
- Больше зарабатывать / второй доход → `pain_11_money`

Если выбрано «Примеры других и окружение» — отправить сообщение про Random Coffee и трекинг-группы (текст в `bot.py`, константа `NETWORK_MSG`).

**Шаг 3 — Q3:** «На какой период дорожную карту?» (одиночный выбор)
- 7 / 14 / 28 / 60 дней

**Шаг 4 — Q4:** «Как получать материалы?» (одиночный выбор)
- Пришли всё сразу → отправить все посты одним сообщением (кнопки Mini App по темам)
- По несколько постов каждые 1-2 дня → drip-рассылка
- По несколько постов + напоминания → drip + напоминания

**После онбординга:**
- Бот собирает посты из `roadmap_all_pains_v4.json` по выбранным в Q2 темам
- Отправляет через кнопку Mini App (`app.html?ids=ID1,ID2,...`)
- Если drip — батчи по 2 поста, интервал зависит от периода (7д→каждый день, 28д→каждые 3 дня)
- Каждые 3 батча спрашивает «Как материалы?» → можно скорректировать через Claude
- Если у пользователя >1 темы — кнопки «Больше про: [тема]» (ещё 5 постов по теме)

**Свободный чат:**
После онбординга пользователь может написать текст → Claude (Haiku) подбирает посты → кнопка Mini App. Двухэтапный поиск: сначала определяет категорию, потом ранжирует.

### 4. Команды
- `/my_roadmap` — показать дорожную карту
- `/pause` / `/resume` — пауза/возобновление drip

## Данные

### Файлы для бота (на сервере Sailbot)
| Файл | Что это |
|------|---------|
| `roadmap_all_pains_v4.json` | 11 тем, для каждой — список постов с ID |
| `knowledge_base.js` | База 496 постов — бот читает для свободного поиска через Claude |

### Файлы для Mini App (уже захощены, менять не нужно)
| Файл | URL |
|------|-----|
| `app.html` | https://olyunyaa.github.io/secrets-library/app.html |
| `knowledge_base.js` | Загружается из app.html |
| `photos/` | Фотографии к постам |

Mini App уже работает. Sailbot просто отправляет кнопку WebApp с URL.

### Создаются автоматически при работе
| Файл | Что это |
|------|---------|
| `user_roadmaps.json` (или таблица в БД) | Состояние drip каждого пользователя |
| `analytics_log.csv` (или таблица в БД) | Лог нажатий кнопок |

## Формат ID постов

Mini App ищет посты по ключу `{channel_id}_{message_id}` (например `2030927165_17`). Этот же формат в `roadmap_all_pains_v4.json`. URL Mini App: `app.html?ids=2030927165_17,2030927165_34,...`

---

## Claude API

Нужен ключ `ANTHROPIC_API_KEY`. Модель: `claude-haiku-4-5-20251001` (~0.01$ за запрос).

Используется для:
- Свободного подбора постов по запросу пользователя
- Коррекции маршрута по фидбэку
- Подбора бонусных постов когда roadmap по теме исчерпан
